What is PushStateTree

A browser history micro-framework that works with events, navigation of single page app easier than ever.

Allows you...

Create awesome easy modular Single Page Application, where you can use all the power of the DOM to make your app fast, with less redrawns, memory leak free, deep-linked, support async changes and will let you create modules with all of their own businness inside, without need to know other pages to keep integrity.

Natively support pushState and History API, with fallback option.

It gives you full control of the URL changes

You can have multiple routers, each router can have rules, what also support sub-rules.

Each rule will give events that you can listen to compose your application and ensure that components related to each rule will be fulfilled in the right sequence order.

Every rule you create will trigger the events:

  • enter: When the rule is match for the first time or after a "leave".
  • change: When the rule is match after a "enter".
  • match: Everytime a location change is dispatch and the rule match.
  • leave: When the rule doesn't match but the previous location was matching the rule.
  • update: Everytime a event is dispatched.
